# Decision Record: agent-002-installer-delivery

## メタデータ
- Status: Accepted
- Date: 2026-04-03
- Scope: インストールおよびブートストラップ導線

## 背景とリサーチ
カスタムエージェントの導入はクリーン環境で簡単に再現できる必要があります。
手作業のコピー手順は反復セットアップでミスと時間ロスを招きます。

リサーチ結果:
- ワークスペースカスタマイズはファイルベースであり、スクリプトで配備可能です。
- Linux/macOS/WSLではシェルインストーラーが最小摩擦の導入方式です。
- 単一エントリのインストーラーで、必要ディレクトリ作成とテンプレート配置を安全に実行できます。

参照:
- https://code.visualstudio.com/docs/copilot/customization/overview
- https://code.visualstudio.com/docs/copilot/customization/custom-agents
- https://code.visualstudio.com/docs/copilot/customization/prompt-files

## 決定
DODエージェント資産を想定パスに配備するためのリポジトリインストーラースクリプトを作成します。

インストーラー責務:
- 実行前提条件の検証
- 必要ディレクトリの作成
- カスタマイズファイルのインストール/更新
- 導入後の確認手順の表示

この親決定の下には、次の現在有効なサブ決定を `DECISIONS.yml` に持ちます。
- テンプレートファイルをインストーラー入力とすること
- 初期CLI対応を GitHub Copilot Chat のみに限定すること

これらのサブ決定の詳細な経緯と理由は、この履歴ファイルに記録します。

## 決定契約
### 不変条件
- インストーラーは冪等であること（複数回実行しても安全）。
- 書き込み先は対象ワークスペースのカスタマイズパスに限定する。
- エラー時は即時失敗し、原因を明示する。
- 生成または配置対象のカスタマイズファイルは、テンプレート資産をソースとする。
- CLIターゲットは明示指定を必須とし、未対応ターゲットは拒否する。

### 非目標
- グローバルなシステムパッケージ導入は行わない。
- ユーザーレベルのVS Code設定を自動変更しない。
- 既存ファイルをサイレントに上書きしない。上書きはユーザーが明示的に許可した場合に限る。
- 初期リリースでの複数CLI同時対応は行わない。

### 受け入れ基準
- 初回実行で必要なDODエージェントファイルをすべて作成できる。
- 再実行時に重複作成や破損が発生しない。
- 出力に成功状態と次の検証アクションが明記される。
- テンプレートと出力先パスが分離され、将来のCLI拡張が可能である。
- `copilot` 指定時に、現在のテンプレート一式を想定ワークスペースパスへ配置できる。
- 対象ファイルが既に存在する場合は、ユーザーが明示的に許可しない限り既存ファイルを維持する。

### 失敗条件
- 再実行で重複設定が発生する。
- 宣言範囲外ファイルを変更する。
- 必要ファイル欠落のまま成功終了する。
- 未対応CLI指定をサイレントで受け入れる。
- 既存対象ファイルをユーザー確認なしで上書きする。

## 調査更新（2026-04-04）
本決定の補強に使用した追加調査結果:
- VS Codeのカスタマイズ資産は `.agent.md`、`.prompt.md`、instructionsファイルなどのMarkdownベースであり、テンプレート駆動の生成/配置に適合する。
- VS CodeのカスタマイズUIは複数のエージェントタイプ（local agents, Copilot CLI, Claude agent）を扱えるため、初期スコープを1CLIに限定する段階導入は妥当。
- カスタムエージェントは background agents（Copilot CLI）でも再利用可能であり、初期はChat限定でも将来拡張を見据えたテンプレート構造の価値がある。

議論結果:
- テンプレートファイル利用ルールは、この履歴ファイルだけに置かず、現在有効なサブ決定として `DECISIONS.yml` に記載する。
- GitHub Copilot Chat限定スコープは、この履歴ファイルだけに置かず、現在有効なサブ決定として `DECISIONS.yml` に記載する。
- テンプレートファイルの実装詳細は重要度が高いため、独立した決定 `agent-004-installer-template-details` として分離する。

## 実装仕様スナップショット（2026-04-10）
- 初期リリースで受け付けるCLIターゲット識別子は `copilot` とする。
- `copilot` 向けの初期マニフェストは次のとおりとする。
	- `templates/agent.md` -> `.github/agents/dod.agent.md`
	- `templates/DECISIONS.yml` -> `DECISIONS.yml`
- インストーラーは、このマニフェストに必要な親ディレクトリを作成する。`.github/agents` を含む。
- 対象ファイルが既に存在する場合、既存ファイルを既定で優先し、上書きが必要ならユーザーに確認する。
- 初期リリースでは、既存ファイル対応のためのバックアップファイルは作成しない。

昇格結果:
- CLIターゲット名、既存ファイル処理ルール、初期テンプレートファイルセットは、現在有効な拘束条件として `DECISIONS.yml` で管理する。

## 影響
Positive:
- 参加者のオンボーディングを安定化できます。
- 環境セットアップのサポートコストを削減できます。

Trade-offs:
- カスタマイズ対象が増えるほどスクリプト保守コストが上がります。
- シェル差異をCIまたは手動で検証する必要があります。

## 実装更新（2026-04-10）
- ルート配置の `install.sh` を実装し、引数なし実行時は `copilot` を既定ターゲットとして進行し、競合時のみ上書き確認するフローにした。
- 公式インストール挙動の安全性と再現性のため、インストーラーの取得元を `mkgask/DODKit@main` に固定した。
- 任意の取得元を指定できる経路（`--repo`、`--ref`）を削除し、任意リポジトリや任意 ref からの資産取得を防止した。
- `install.sh` を source してパーサ、ターゲット検証、コピー挙動を確認する関数単位テスト `tests/install.test.sh` を追加した。
